# v0.1.124 - 修复自动全屏不横屏

## 更新内容

- 修复手机旋转后播放器只进入竖屏全屏、没有真正变成横屏的问题。
- 播放器内嵌状态继续允许竖屏显示，全屏状态切换为仅支持横屏方向。
- 自动横屏全屏弹出后再次向系统请求 landscape 几何方向，确保界面真实旋转。
- 手动点击系统全屏按钮时也同步切换为横屏-only，避免手动路径仍停留竖屏。
- 退出全屏后恢复内嵌播放器的竖屏/横屏兼容方向，非播放器页面继续保持竖屏。
- 保持同一个 AVPlayer 实例，横竖屏切换不重建播放对象，播放进度不丢失。

## 打包说明

- GitHub Actions 会构建 Release 配置的未签名 IPA。
- 生成的 `ccbili-unsigned.ipa` 会作为 Release 附件上传。
- 未签名 IPA 需要自行重签名后安装到真机。

## 验证建议

- 竖屏进入视频页并开始播放。
- 直接把手机旋转到横屏，确认播放器进入横屏全屏，而不是竖屏铺满。
- 横屏左右两个方向都测试一次，确认方向能跟随设备。
- 旋回竖屏，确认自动退出全屏并回到内嵌播放器。
- 再点击系统全屏按钮进入全屏，确认手动路径也是真横屏。
