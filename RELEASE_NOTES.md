# v0.1.128 - 修复横屏切换画面错位

## 更新内容

- 修复旋转横屏时出现“竖屏详情页还在、播放器画面先横屏/错位”的过渡问题。
- 横屏前只先放开播放器需要的方向支持，不再提前请求整个页面旋转，避免导航栏和详情页参与播放器全屏动画。
- 全屏弹出改为从当前窗口根控制器发起，减少内嵌播放器子控制器参与转场导致的层级错乱。
- 视频详情页和首页的竖屏锁增加播放器全屏保护，播放器正在全屏时不会抢回竖屏。
- 外层 SwiftUI 播放器在全屏过渡触发 `onDisappear` 时不再暂停播放，避免旋转时播放中断或进度卡顿。

## 原因说明

之前横屏自动全屏时，代码在 `AVPlayerViewController` 真正弹出前就请求整个 Scene 横屏，同时视频详情页的消失逻辑又可能把方向锁回竖屏。系统转场期间就会短暂出现竖屏页面、横屏播放器层、黑边同时存在的混合状态。现在改成先让播放器全屏状态接管方向，再由 AVKit 全屏完成后请求横屏几何更新。

## 打包说明

- GitHub Actions 会构建 Release 配置的未签名 IPA。
- 生成的 `ccbili-unsigned.ipa` 会作为 Release 附件上传。
- 未签名 IPA 需要自行重签名后安装到真机。

## 验证建议

- 竖屏进入视频详情页并开始播放。
- 旋转手机到横屏，确认不会再看到竖屏详情页和横屏播放器叠在一起。
- 横屏播放期间确认视频不暂停、进度不跳动。
- 旋回竖屏，确认回到内嵌播放器且继续播放。
- 连续横竖屏切换多次，确认没有重复 overlay、卡住或黑屏。
