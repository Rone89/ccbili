# v0.1.146 - 详情页性能、评论布局与播放器控件优化

## 更新内容

- 统一视频详情页信息卡、评论入口、简介推荐三个主容器的液态玻璃质感，圆角和玻璃着色保持一致。
- 评论弹窗改为原生 List 逐行渲染，布局参考 Bilibili 评论结构：头像、用户名、正文、时间、点赞和明确的“查看回复”入口。
- 修复点击“查看回复”误触用户主页的问题：评论弹窗内头像和昵称不再承担导航点击，避免和回复按钮抢事件。
- 删除用户主页下拉刷新，只保留进入页面自动加载和列表底部加载更多。
- 优化整体滑动性能：评论列表不再作为单个大视图渲染，相关推荐限制首屏数量，首页/搜索/用户页封面解码尺寸降低以减少 CPU、内存和发热。
- 修复竖屏播放器原生暂停、进度条等控件不显示的问题：竖屏内嵌状态下弹幕层和手势层不再拦截 AVPlayerViewController 的系统控件触摸，横屏全屏仍保留手势层。

## 验证建议

- 打开视频详情页，确认三个主容器玻璃效果统一，信息卡、评论入口、简介推荐视觉一致。
- 点击“查看评论”，确认评论弹窗顶部不覆盖视频区域，评论行布局清晰，用户名白色、正文灰色。
- 在评论弹窗点击“查看 X 条回复”，确认只打开回复弹窗，不跳转用户主页。
- 打开用户主页，下拉页面确认不会触发刷新动画。
- 快速滑动首页、搜索结果、详情推荐和评论弹窗，观察掉帧和发热是否降低。
- 竖屏播放视频时点击播放器，确认系统暂停按钮、进度条和全屏按钮都能显示并可操作；横屏全屏时手势仍可用。
