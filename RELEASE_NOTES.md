# v0.1.151 - 统一详情玻璃、评论反色与 1080P+ 恢复

## 更新内容

- 详情页信息卡、评论入口、简介推荐三个主容器统一为评论入口同款 Liquid Glass 效果，全部使用交互态 `.regular.interactive()`。
- 修复浅色背景下评论/回复文字不反色：用户名改用 `.primary`，正文改用 `.secondary`，随系统明暗和玻璃背景自动适配。
- 修复退出 App 或锁屏一段时间后 1080P+ 无法加载：播放源缓存加入登录 Cookie 指纹，避免游客降级播放源污染登录态高码率缓存。
- 回到前台时同步 B 站 Cookie、刷新登录状态，并在播放源过旧或登录态变化时强制重拉播放源，避免继续使用过期 CDN 地址。
- 1080P+ 请求前会先恢复持久 Cookie，减少锁屏/后台后因共享 Cookie 暂空导致的清晰度降级。

## 验证建议

- 打开视频详情页，确认三张主容器玻璃效果和“查看评论”入口一致。
- 在浅色视频背景下打开评论和回复，确认用户名为深色主文本、正文为灰色副文本，不再白字看不清。
- 登录后选择 1080P+，退出 App 或锁屏 1 分钟以上再回来，确认仍能重新加载 1080P+ 或正确显示可用清晰度。
- 如遇 B 站账号权限不足，仍会按现有逻辑自动降级到可播放清晰度。

## 打包说明

- GitHub Actions 会构建 Release 配置的未签名 IPA。
- 生成的 `ccbili-unsigned.ipa` 会作为 Release 附件上传。
- 未签名 IPA 需要自行重签名后安装到真机。
