# v0.1.127 - 横竖屏复用同一个播放器

## 更新内容

- 修复横竖屏切换时播放进度丢失的问题。
- AVKit 播放路径改为同一个 `AVPlayer` + 同一个 `AVPlayerViewController`，横竖屏只搬移容器，不新建播放器控制器。
- 竖屏内嵌播放和横屏全屏播放共享同一个播放实例，避免切换时重新挂载导致进度/状态抖动。
- 全屏期间禁止 SwiftUI 更新把播放器重新塞回竖屏容器，避免双容器抢同一个播放器。
- 弹幕层、手势层仍挂在同一个 `contentOverlayView`，随播放器控制器一起移动。
- 退出全屏后再把同一个播放器控制器挂回竖屏内嵌容器，播放状态和倍速继续保留。

## 原因说明

之前虽然复用了 `AVPlayer`，但横屏时会创建另一个 `AVPlayerViewController`，竖屏容器和横屏容器在切换期间可能同时争抢同一个 player/overlay。系统在重新挂载播放输出时会出现短暂停顿，严重时进度状态会看起来丢失。现在改成同一个播放器控制器在内嵌和全屏之间搬家，切换时不重建播放输出。

## 打包说明

- GitHub Actions 会构建 Release 配置的未签名 IPA。
- 生成的 `ccbili-unsigned.ipa` 会作为 Release 附件上传。
- 未签名 IPA 需要自行重签名后安装到真机。

## 验证建议

- 竖屏播放到任意进度后旋转横屏，确认进度不跳回、不丢失。
- 横屏播放一段时间后旋回竖屏，确认继续从当前进度播放。
- 连续横竖屏切换多次，确认不会暂停、不会重新加载视频。
- 检查弹幕/手势层在横屏和竖屏都仍然可用。
